🛡️ ERROR HANDLING
═══════════════════════════════════════════════════════════════════════

1. BMKG API FAILURE
   Problem: API down, timeout, rate limit
   Solution: Auto-fallback to default data
   Result: Dashboard tetap berfungsi, tidak crash
   
   Default Data:
   - Temperature: 28°C
   - Humidity: 70%
   - Weather: "Data tidak tersedia"
   - Rainfall: 0mm
   - Wind: 5 km/h

2. NO ACTIVE DEVICES
   Problem: Semua node offline atau tidak ada device
   Solution: Return empty summary + BMKG data
   Result: Metrics = 0, status = "No Data", BMKG tetap muncul

3. DATABASE FAILURE
   Problem: Query timeout, connection lost
   Solution: Cache layer masih serve stale data
   Result: Graceful degradation, user tetap dapat info
