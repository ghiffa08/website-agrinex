STEP 1: START
  User → Klik "Kalibrasi AI" 
       → Klik "Mulai Kalibrasi"
       → Status: idle → user_saturating

STEP 2: SATURATE
  User → Siram tanah hingga jenuh air
       → Konfirmasi "Saturasi Selesai"
       → Status: user_saturating → waiting_24h
       → System: Simpan timestamp saturasi

STEP 3: WAIT & COLLECT
  System → Collect sensor data 24-72 jam
         → Monitor drainase gravitasi
         → Status: waiting_24h

STEP 4: ANALYZE (Auto via Cron)
  System → Cron job setiap 2 jam
         → Check if ready (min 24h passed)
         → Fetch sensor data (aggregate per jam)
         → Send to Gemini API
         → Parse AI response
         → Status: waiting_24h → analyzing → completed/waiting_24h

STEP 5: RESULTS
  User → View hasil di modal
       → FC ADC value
       → Confidence score
       → AI reasoning & recommendations
       → Status: completed


🧠 AI ANALYSIS LOGIC
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

INPUT DATA FORMAT (Optimized for Token Efficiency):
{
  "device_id": 65,
  "saturation_timestamp": "2026-07-23T08:00:00Z",
  "analysis_hours": 24,
  "data_points": 24,
  "hourly_readings": [
    {
      "timestamp": "2026-07-23T09:00:00",
      "soil_moisture_avg": 85.5,
      "soil_adc_avg": 1245,
      "soil_adc_min": 1230,
      "soil_adc_max": 1260,
      "temperature_avg": 28.3
    }
    // ... max 72 data points (3 hari)
  ]
}

AI TASKS:
1. Deteksi saturasi point (ADC terendah setelah penyiraman)
2. Deteksi drainase gravitasi (ADC naik lalu stabil)
3. Identifikasi field capacity (ADC saat stabil 24-48h setelah saturasi)
4. Deteksi wilting point (ADC tertinggi sebelum saturasi) - optional
5. Hitung confidence score berdasarkan pola clarity

OUTPUT FORMAT:
{
  "field_capacity_adc": 1650,
  "wilting_point_adc": 2750,
  "confidence_score": 82.5,
  "saturation_detected_at": "2026-07-23T08:15:00Z",
  "fc_detected_at": "2026-07-24T09:00:00Z",
  "analysis_quality": "good",
  "reasoning": "Pola drainase jelas terdeteksi...",
  "recommendations": ["Lanjutkan monitoring 24 jam untuk akurasi lebih tinggi"]
}

PROGRESSIVE LEARNING:
├── Iteration 0 (24h):  Confidence max 70%  → "Need more data"
├── Iteration 1 (48h):  Confidence max 85%  → "Good enough" ✅
└── Iteration 2 (72h):  Confidence max 95%+ → "Excellent" ✅✅

🔌 API ENDPOINTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

POST   /api/v1/devices/{deviceId}/ai-calibration/start
       → Mulai proses kalibrasi
       → Response: { success, message, status, started_at }

POST   /api/v1/devices/{deviceId}/ai-calibration/confirm-saturation
       → User konfirmasi saturasi selesai
       → Response: { success, message, status, saturation_at }

POST   /api/v1/devices/{deviceId}/ai-calibration/analyze
       → Trigger analisis manual (jika sudah 24h+)
       → Query: ?force=true (untuk force re-analysis)
       → Response: { success, message, status, results }

GET    /api/v1/devices/{deviceId}/ai-calibration/status
       → Get current calibration status
       → Response: { success, data: { status, iteration, can_analyze, ... } }

POST   /api/v1/devices/{deviceId}/ai-calibration/cancel
       → Cancel ongoing calibration
       → Response: { success, message }
