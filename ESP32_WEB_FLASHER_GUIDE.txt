📋 OVERVIEW
═══════════════════════════════════════════════════════════════════════════════

ESP32 Web Flasher adalah fitur untuk flash firmware NODE01-Sender, NODE02-Receiver,
dan Node Tester langsung dari browser tanpa perlu install software desktop seperti
PlatformIO, Arduino IDE, atau Python.

TEKNOLOGI:
  ✓ WebSerial API (Chrome, Edge, Opera)
  ✓ ESPTool.js (JavaScript port dari esptool.py)
  ✓ Laravel Blade + Bootstrap 5
  ✓ ESP Web Tools manifest format

FITUR:
  ✓ Flash 3 jenis firmware (Sender, Receiver, Tester)
  ✓ Real-time progress bar dengan speed indicator
  ✓ Serial monitor terintegrasi
  ✓ Auto-detect ESP32 via USB
  ✓ Konfigurasi WiFi untuk Receiver
  ✓ Responsive UI dengan neumorphic design

🚀 CARA MENGGUNAKAN (USER)
═══════════════════════════════════════════════════════════════════════════════

1. AKSES WEB FLASHER:
   https://smartdrip-system.agrinex.io/admin/flasher

2. REQUIREMENTS:
   ✓ Browser: Chrome, Edge, atau Opera (support Web Serial API)
   ✓ ESP32 board: Seeed XIAO ESP32-S3
   ✓ USB Cable: USB-C to USB-A/C
   ✓ Role: Admin atau Operator

3. STEP-BY-STEP:
   
   a. Hubungkan ESP32 ke laptop via USB-C
   
   b. Pilih firmware dari dropdown:
      - NODE01-Sender: LoRa TX dengan sensor
      - NODE02-Receiver: LoRa RX dengan WiFi sync
      - Node Tester: Diagnostics tool
   
   c. (Optional) Isi konfigurasi WiFi jika pilih Receiver:
      - WiFi SSID
      - WiFi Password
      - API Key (auto-filled dari server)
   
   d. Klik tombol "Flash Firmware"
   
   e. Browser akan minta akses serial port:
      - Pilih port ESP32 (biasanya "USB Serial" atau "ttyACM0")
      - Klik "Connect"
   
   f. Tunggu proses flashing (~30-60 detik):
      - Progress bar akan menunjukkan status
      - Speed indicator menunjukkan kecepatan transfer
      - Log serial akan muncul saat device restart
   
   g. Setelah selesai:
      - Device akan restart otomatis
      - Cek serial monitor untuk verifikasi boot berhasil
      - LED onboard akan berkedip jika firmware berjalan

4. TROUBLESHOOTING:
   
   ✗ Port tidak muncul?
     → Pastikan driver USB-Serial ter-install
     → Coba cabut-pasang USB
     → Coba port USB lain
   
   ✗ Flash gagal "Failed to connect"?
     → Hold BOOT button saat klik flash
     → Reset ESP32 dengan menekan RST button
     → Pastikan baud rate 115200
   
   ✗ Browser warning "Web Serial not supported"?
     → Gunakan Chrome/Edge/Opera versi terbaru
     → Firefox & Safari belum support Web Serial API

🔧 CARA REBUILD FIRMWARE (DEVELOPER)
═══════════════════════════════════════════════════════════════════════════════

Jika ada perubahan source code firmware, rebuild dengan script:

1. PREREQUISITES:
   ✓ PlatformIO CLI installed (pip install platformio)
   ✓ Python 3.7+
   ✓ Source code NODE01-Sender, NODE02-Receiver, agrinex_node_tester

2. RUN BUILD SCRIPT:
   ```bash
   cd /path/to/Projects
   python3 build-flasher-firmware.py
   ```

3. SCRIPT AKAN:
   ✓ Clean previous build
   ✓ Build semua firmware via PlatformIO
   ✓ Copy binaries ke public/flasher-firmware/
   ✓ Generate manifest.json untuk ESP Web Tools
   ✓ Generate README.md

4. OUTPUT:
   ```
   ======================================================================
   Build Summary: 3/3 successful
   ======================================================================
   
   ✓ All firmware built successfully!
   
   Firmware location: .../agrinex-smartdrip/public/flasher-firmware
   
   Next steps:
   1. Commit dan push ke repository
   2. Deploy ke production server
   3. Akses Web Flasher
   ```

5. DEPLOY KE PRODUCTION:
   ```bash
   cd agrinex-smartdrip
   git add public/flasher-firmware
   git commit -m "update: rebuild ESP32 firmware binaries"
   git push origin main
   
   # SSH ke production
   ssh user@production-server
   cd /path/to/web
   git pull origin main
   ```

🌐 BROWSER COMPATIBILITY
═══════════════════════════════════════════════════════════════════════════════

SUPPORTED (Web Serial API):
  ✅ Chrome 89+ (Desktop: Windows, Mac, Linux)
  ✅ Edge 89+ (Desktop: Windows, Mac)
  ✅ Opera 75+ (Desktop: Windows, Mac, Linux)

NOT SUPPORTED:
  ❌ Firefox (Web Serial API not implemented yet)
  ❌ Safari (Web Serial API not supported)
  ❌ Mobile browsers (iOS/Android - API not available)

FALLBACK:
  - Jika browser tidak support, warning message ditampilkan
  - User diarahkan untuk gunakan Chrome/Edge/Opera
  - Desktop Jig (Python app) tetap tersedia sebagai alternative
